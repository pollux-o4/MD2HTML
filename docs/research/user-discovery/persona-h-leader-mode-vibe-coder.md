# 박서윤 — md-show-me 페르소나 분석

## 1. 안녕하세요, 박서윤입니다

저는 27살이고, 학부 때는 생물학 했는데 졸업 무렵에 코딩이 재밌어서 부트캠프로 풀스택+AI 수업 들었어요. 작년에 스타트업에서 인턴 6개월 — 백엔드부터 프론트, 앱까지 다 했고, 무엇보다 기획팀이랑 개발팀 사이에서 의견 조율하고 PR 머지하는 *bridge* 역할이 제일 재밌었어요. 지금은 취준 중이고, 그 사이에 사이드 프로젝트랑 팀 프로젝트 같이 굴리고 있습니다.

## 2. 한 주 흐름 — .md 가 쌓이는 패턴

월요일: 사이드 프로젝트 새 feature 기획 — Claude Code 한테 "이 부분 리서치 sub-agent 로 돌려줘" 함. 결과 .md 3개 생김. 화요일: 팀 프로젝트 미팅 노트, ADR 1개. 수요일: 부트캠프 복습 — 학습 .md 를 Obsidian 에 정리, 백링크 연결 시도. 목요일: 취준 — 회사별 기술 스택 분석 .md. 금요일: 사이드 프로젝트 dev log + 다음 주 plan.

한 주에 .md 가 10~15개씩 쌓이는데, *내가 다시 여는 건 절반도 안 됨*. Obsidian 백링크 그래프는 점점 빽빽해지는데 어디서 어디로 흐르는지 한눈에 안 들어와요. GitHub repo 의 `docs/` 도 마찬가지 — 새 파일 만들 때마다 "이거 어디 폴더에 둬야 하지" 고민이 늘어요.

## 3. AI 글 피로 — 6가지 pain 구체화

(1) **Sub-agent verbose + 종합 안 됨** — 가장 큼. 리서치 sub-agent 돌리면 5000 단어짜리 .md 가 3개 떨어지는데, 정작 *"그래서 결론이 뭔데"* 가 없음. 종합은 내가 다시 해야 함. (2) **학습 .md 가 너무 김** — 부트캠프 복습용으로 만든 .md 가 한 페이지 안에 안 들어와서 스크롤하다 맥락 놓침. (3) **백링크 파악 불가** — Obsidian 그래프 view 켜도 노드 50개 넘으면 의미 없는 거미줄. *어느 노트가 어느 결정의 부모인지* 안 보임.

(4) **의사결정 시 종합 부재** — 사장 mode 로 "이 PR 머지할까 말까" 결정하려는데, 관련 ADR + 코드 diff + sub-agent 리서치 결과를 *나란히 펼친 view* 가 없음. 일일이 탭 열어야 함. (5) **정보 위계 결핍** — "학교 → 학생 → 노트 → 목차" 같은 자연스러운 위계가 .md 더미에는 없음. 다 평면 (flat). (6) **AI 의 영한 무분별 혼용** — Claude 가 "trace 떠보면 verification 이 fail 했고..." 식으로 쓰는데, 한글 문맥에 영어 단어가 음차 없이 박혀서 읽다 멈춤. *코드 안의 영어는 OK*, 한글 문장 안의 의미 없는 영한 혼용은 거부.

## 4. seed 가설 검증/반박 — 페르소나 H 입장

**"AI 글 지친 사람"** — 검증. 단 박지훈/Maya 와 결이 다름. 저는 *학습자 + 생산자* 양쪽. 학습용 .md 가 너무 verbose 해서 학습 효율 떨어지고, 생산한 .md 는 다시 안 봐서 의사결정 시 못 씀.

**YouTube/PPT** — 반박. 저는 *베스트셀러 책 톤* 이 ideal. 도메인 무관하게 readable + 전문 용어는 일상 예시로 풀이 + 위계가 살아있는 책 (예: 사피엔스, 팩트풀니스). Stratechery 보단 책, PPT 보단 잘 짜인 챕터 구조.

**"정리된 거 보고 싶다"** — 검증. 단 *능동 학습* 욕구가 강해서 정적 요약만으론 부족. 인터랙티브 (펼치고 접고, 백링크 따라가기, 게임적 요소) 가 같이 있어야 함. 단 범용 도구라는 점을 인정 — 정적 시각도 같이 제공돼야 다른 사용자도 씀.

## 5. md-show-me ideal scenario 3개 — 사장 mode 의사결정

- `/show-me 이번 주 sub-agent 리서치 3개 종합해서 1페이지로 — 결론 + trade-off + 내가 결정해야 할 포인트만` — 사장 mode 의사결정용. 4분 안에 펼쳐지면 즉시 머지/리젝 결정.
- `/show-me 부트캠프 학습 .md 들을 "학교→학생→노트→목차" 위계로 재배열 + 백링크 그래프를 의미 단위로 묶어줘` — 학습 navigation. 평면 더미를 위계 있는 인터랙티브 view 로.
- `/show-me 이 ADR 의 영한 혼용 단어들 한글 풀이 + 일상 예시로 다시 써줘 (베스트셀러 톤)` — 팀원 공유용. 비전공 PM 도 읽을 수 있게.

## 6. unmet needs

**Claude Code 빈틈** — sub-agent 가 verbose 결과 던지고 끝. *종합 layer* 가 없음. 사장 mode 사용자는 부서 엔지니어링 직접 안 하니, 종합 안 된 raw output 은 무용지물.

**Obsidian 빈틈** — 백링크 자체는 있는데 *위계가 없음*. 그래프 view 는 노드 많아지면 무의미. 의미 단위 grouping 이 필요.

**GitHub 빈틈** — `docs/` 폴더는 평면. .md 끼리의 *결정 lineage* (이 ADR 이 저 PR 의 부모) 가 안 보임.

빈 공간 = **"sub-agent + Obsidian + GitHub 의 .md 들을, 위계 있는 베스트셀러 톤 인터랙티브 view 로 종합해서 사장 mode 의사결정을 돕는 layer"**.

## 7. 다른 9 페르소나 와의 unique 차별점

- vs **박지훈** (시니어 백엔드): 박지훈은 CS 전공 + 10년차. 저는 비전공 + 취준. 대신 인턴에서 *기획-개발 bridge 경험* — 박지훈은 코드만 봄, 저는 양쪽 다 봄.
- vs **Maya** (US 풀스택 주니어): Maya 는 US, 영어, 코드 중심. 저는 한국, 풀스택 + *기획 가능*, 한글 위주.
- vs **김민호** (tech lead): 같은 사장 mode 지만 김민호는 시니어 + 직장인. 저는 *비전공 + 취준* — 학습 비중이 훨씬 큼.
- vs **김도윤** (indie hacker): 김도윤은 사업/운영 product 중심. 저는 사업 X, *학습 + 팀 프로젝트* 비중.
- vs **이수민** (학생 vibe-coder): 이수민은 CS 부전공 X, 학부생. 저는 풀스택 인턴 + bridge 경험으로 *더 mature*.

**페르소나 H 의 unique** = *bridge 경험 + 비전공 + 취준 + 사장 mode + 능동 시각 학습* 의 교집합. 다른 9명 중 누구도 이 5개를 동시에 갖지 않음.

## 8. 디자인 선호 — 베스트셀러 톤 + 정보 위계 + 친근 명확

- **베스트셀러 책 톤** 기본선. 사피엔스/팩트풀니스의 *전문 용어 → 일상 예시* 풀이. Stripe Press / Linear 톤도 OK (적당히 editorial).
- **마케팅 톤 거부**. "혁신적인 솔루션" 같은 문구 즉시 거부. 친근 + 명확 — 팀원한테 카톡 보내듯.
- **정보 위계 필수**. 학교→학생→노트→목차 식의 자연 위계. 평면 더미 X.
- **한글 위주 + 코드/AI prompt 는 영어**. 영한 자연 코드 스위치 OK, 한글 문장 안 영한 음차 무분별 혼용 거부.
- **인터랙티브 default + 정적 시각도 제공**. 펼치고 접고, 백링크 따라가기. 단 범용 도구이므로 정적 view 도 fallback.
- **공유 시나리오**: 개인 dogfood 60% + 팀원 40%. 팀원이 비전공이어도 읽을 수 있는 친근·명확 톤.

## 9. v1 primary 자격 근거

페르소나 H = **v1 primary umbrella 의 핵심**. 다른 페르소나 (시니어 dev, 디자이너) 는 v1.x adjacent, 비개발자 (서연/지수) 는 v2 expansion. H 는 *사용자 본인이 dogfood 가능* → 빌드-사용-피드백 loop 가 최단. bridge 경험 덕에 기획·개발 양면을 한 사람이 검증 가능 — 다른 페르소나는 한쪽만 봄.
