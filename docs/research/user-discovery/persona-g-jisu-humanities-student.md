# 박지수 (23, 사회학과 4학년)

안녕하세요 박지수예요. 사회학과 4학년이고 졸업논문 (글, 50p 분량) 쓰는 중이에요. PPT 는 *졸논 외* 가 더 많아요 — 마케팅 인턴쉽 주간 발표, 조별과제 3개, 학회 학부생 세션 1개. 한 학기에 PPT 6~8장은 만드는 것 같아요. 졸논 발표는 학기 끝에 1회뿐. 코드는 진짜 1도 몰라요. GitHub? 들어본 적은 있는데 뭐 하는 덴지 몰라요. README 이런 단어도 처음 들어요. 근데 ChatGPT 는 거의 매일 써요. 인턴쉽 시작하고는 Perplexity 도 가끔 쓰고요. 이걸 깔고 보고 적을게요.

## 1. 한 주 정보 흐름 — 조사 → 정리 → PPT

평일 낮은 인턴쉽, 저녁은 조별과제, 주말은 졸논. 졸논은 *글* 이라 통째로 Word 에서 작업하고, PPT 는 인턴쉽 + 조별과제 + 학회 발표 쪽이 매주 1~2장씩 깔려요. 월요일에 교수님이 "다음 주까지 이 주제 자료 조사해 와" 하면 일단 ChatGPT 한테 "사회자본 이론 핵심 논문 5개 정리해줘" 던져요. 그러면 길게 답이 나오는데 *대부분 못 써요*. 출처가 가짜인 경우가 많아서 결국 Perplexity 로 다시 던지거나 RISS 가서 진짜 paper 찾고, 초록만 ChatGPT 한테 복붙해서 "한 문단으로 요약" 시켜요.

그 결과물을 Notion 페이지 하나 만들어서 쌓아요. 근데 Notion 이 *끝없이 길어져요*. 3주 지나면 스크롤 한참 내려야 하고 뭐가 어디 있는지 까먹어요. 그 다음 PPT 만들 때가 진짜 문제예요. Notion 의 한 문단을 슬라이드 1장에 옮기려고 하면 *글자가 너무 많아요*. ChatGPT 가 쓴 "이러한 연구는 ~할 수 있다는 점에서 의의가 있다고 할 수 있다" 같은 문장을 슬라이드에 넣으면 진짜 답답해 보여요. 그래서 매번 직접 "사회자본 → 신뢰 → 협력 ↑" 이런 식으로 *제가 다시 줄여요*. 이 과정이 제일 오래 걸려요.

## 2. AI 글 피로 — 조사 결과로 받은 글의 어색함

한글 AI 글, 진짜 *번역체* 티 너무 나요. "당사는", "할 수 있습니다", em dash (—) 막 끼어 있는 거. 교수님이 "이거 AI 로 쓴 거야?" 한 번 물어보신 적 있어서 그 뒤로 *표절 검사기랑 GPT 검출기* 두 번 돌려요. 통과해도 불안해요. 그래서 조사용으로는 쓰는데 *그대로 paste 는 절대 안 해요*. 항상 제 말로 다시 풀어 써요.

PPT 옮길 때도 똑같아요. AI 가 쓴 한 문단을 슬라이드에 그대로 넣으면 *15줄 짜리 글덩어리* 가 돼요. 발표할 때 청중이 슬라이드 읽느라 제 말 안 들어요. 그래서 슬라이드 1장 = bullet 5~7개, 한 줄 12~15자 정도로 끊는 게 제 룰이에요. AI 글은 이 룰이랑 정반대로 만들어져 있어요.

## 3. seed 가설 검증/반박

*"AI 글 지친 사람"* — 네 저 맞아요. 근데 저는 매끈한 글이 싫은 게 아니라 *그 매끈한 글을 슬라이드에 못 옮긴다* 가 더 정확해요. 길고 풍부한 건 *읽을 때는* 좋아요. *발표 자료로 쓰려면* 짧고 시각적이어야 해요.

*"YouTube / PPT 비유"* — YouTube 비유는 좀 멀어요. 저는 긴 YouTube 잘 안 봐요. PPT 는 *제가 만드는 쪽* 이라 reference 라기보다 *산출 포맷*. 진짜 native reference 는 — **짧은 social (Insta reels / 유튜브 shorts / 트윗 thread)** 이 일상 소비, **Wikipedia** 가 조사용 1차, **editorial (브런치 essay)** 가 가끔 긴 글 읽고 싶을 때. 데이터 viz (NYT Upshot) 는 인스타에서 흘러오면 봄. 슬라이드 1장 = 한 메시지, 5~7 bullet, 한 줄 짧게 — 이 *PPT 룰* 이 곧 *Shorts / Reels 룰* 과 같아요. hook → 핵심 → outro.

*"정리된 거 보고 싶다"* — 저한테 "정리됐다" = *한 화면에 핵심이 다 보임*. Notion 처럼 스크롤해야 하면 *정리 안 된 거* 예요. PPT 1장에 들어갈 만큼이 정리의 단위예요.

## 4. md-show-me ideal scenario — *만약 비개발자 친구가 추천했다면*

솔직히 .md 가 뭔지 몰라요. GitHub 에서 글 본 적도 없고요. 근데 *친구가 "이거 발표 자료 만들 때 좋아"* 하면서 웹 서비스로 알려주면 써볼 것 같아요. 시나리오:

- **이번 주 paper 5개 한 페이지 요약** — 졸업논문 조사한 거 한 화면에 모아주면 지도교수님 면담 갈 때 화면 띄워놓고 설명하기 좋아요.
- **인턴쉽 발표 자료 outline** — 회사 공식 자료 던지고 "5분 발표 슬라이드 outline" 뽑아주면 그걸 Google Slides 로 옮길 수 있어요.
- **조별과제 자료 한 페이지** — 팀원 4명이 각자 보낸 docx / 링크 / 캡처를 하나로 모아 *한 화면에 누가 뭘 했는지* 보여주면 발표 전날 새벽 3시에 진짜 살아요.

근데 *github / md / repo* 라는 단어가 입구에 보이면 바로 뒤로 가요. 진입 장벽 핵심은 *내 자료를 어떻게 넣느냐* 예요. 드래그앤드롭이면 OK, "repo clone" 이러면 끝.

## 5. unmet needs

ChatGPT 는 PPT 안 만들어줘요. 만들어도 못생겼어요. Microsoft Designer / Beautiful.AI 써봤는데 *템플릿 갇힘* 느낌이고 한글 폰트가 이상해요. Canva 는 이쁜데 *내 조사 내용을 다시 다 타이핑* 해야 해서 결국 안 써요. Notion 은 발표용 화면이 아니에요.

*AI 조사 결과 → 발표용 시각 산출* 사이가 통째로 빈 칸이에요. 지금은 *제가 손으로* 그 다리를 놓고 있어요. 매주 4~5시간씩.

## 6. 디자인 선호 — 인문계 학생 미적 기준

브런치 / Substack / Wikipedia 같은 *editorial* 톤 좋아해요. 학교 PPT 의 알록달록한 거 (파랑 배경 + 노란 글씨 + 그림자 + WordArt) 진짜 싫어요. 근데 회사 IR 자료 같은 *corporate 파랑* 도 차가워서 싫어요. 그 중간, *논문 표지 같은 절제된 흑백 + 1~2 포인트 컬러* 가 제일 좋아요.

폰트는 Pretendard, KoPubWorld 같은 *얇고 깔끔한 한글* 좋아해요. 명조도 OK (브런치 톤). 다크모드는 별 관심 없어요. 학교 노트북 light 기본이고 발표는 빔프로젝터라 어차피 light 가 보기 편해요.

도구가 결과물을 *제 발표 자료의 reference 톤* 으로 뽑아주면 그걸 보면서 PPT 만들 때 "아 이런 느낌으로 가야지" 기준이 생겨요. 그게 제일 큰 가치일 것 같아요.
