## 2. AI 글 피로 — 학생 입장에선 좀 다름

다른 페르소나들은 *"AI 글 verbose 해서 지친다"* 라는데, 나는 좀 결이 다름. 나는 **AI 글을 안 읽음**. 애초에. 내 README 도 AI 가 써준 거 그대로 push 하고 안 봄.

## 5. unmet needs

- **GitHub README 가 portfolio 용으로 안 어울림.** 톤이 개발자스럽고 딱딱함. 학생이 *"내가 이거 만들면서 이런 거 배웠어요"* 보여주기엔 미스매치.
- **Notion / Behance 의 코드 영역 빈약.** Behance 는 *디자인 작업물* 위주. Notion 은 *문서* 위주. 학생 코드 사이드프로젝트 전용 sharing layer 가 없음.
- **Z세대 sharing culture vs dev 도구의 mismatch.** 우리는 Insta / X / Discord 에 *스샷 1장* 으로 공유함. 근데 dev 도구들은 *링크 → 클릭 → 페이지 로드 → 읽기* 를 요구. friction 너무 큼.
- **"이거 뭐 만들었지" 망각 문제.** 1주일만 지나도 까먹음. AI 가 *내가 뭐 만들었는지* 자동 요약해주면 진짜 좋겠음.
