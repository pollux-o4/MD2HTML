▾ ADR-008 — Performance Tuning Baseline

Context: decimal.js 도입 (ADR-001) 이후 평균 식 평가 2ms → 21ms. UI 끊김.

Decision: budget 단순식 <5ms / 복잡식 <50ms. 매 PR 벤치 자동 회귀. 핫패스 fast-path (네이티브 number).

Consequences: 21ms → 7ms, fast-path 분기 증가, 일부 edge case 정밀도 손실.

▸ ADR-009 — Expression Caching Strategy

Context: 반복 식 매번 재계산.

Decision: LRU AST 캐시(256) + 결과 캐시(128). 변수 바인딩 변경 시 결과만 무효화.

Consequences: 반복 식 7ms → 0.2ms. +2MB 상한. panic 복구 (ADR-007) 와 무효화 상호작용 주의.

▸ ADR-010 — Lazy Evaluation

Context: if(cond, then, else) 가 양쪽 분기 모두 평가.

Decision: if/and/or short-circuit. 사용자 함수 인자는 기본 eager. lazy(...) 명시 가능.

Consequences: 조건식 35% 개선. thunk 도입으로 디버깅 난이도 ↑. 캐시 키와 상호작용.
