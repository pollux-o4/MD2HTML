ADR 0003

Zero Division Handling

Accepted - 2026-03-14

Context

계산기 프로젝트에서 0으로 나누기를 IEEE 754 기본 동작 (Infinity / NaN) 대신 어떻게 처리할지 결정. 회계/재무 용도라 조용한 잘못된 값 전파가 치명적. 사용자 인터뷰 (n=12) 중 8명이 "Infinity 표시는 버그로 인식".

대안 검토 (4개)
  1. NaN 유지 — 호환성 최고, UX 최악
  2. null 반환 — 타입 시그니처 오염
  3. Result<T,E> 패턴 — JS 생태계 부적합
  4. 예외 throw — 가장 명시적 (채택)

Decision

NaN 반환 대신 DivisionByZeroError 예외를 던진다.

Before

function divide(a, b) {
  return a / b;
  // 1/0 -> Infinity
  // 0/0 -> NaN
}

After

function divide(a, b) {
  if (b === 0) {
    throw new DivisionByZeroError(a, b);
  }
  return a / b;
}

Consequences

Positive

  • 사용자가 에러 명확히 인지 (Infinity 표시 사라짐)
  • try/catch 로 의도적 처리 가능
  • 타입 안전 (정상 경로는 항상 number)
  • 디버깅 정보 풍부 (피연산자 포함)

Negative

  • 기존 NaN 검사 코드 14곳 수정 필요
  • throw 비용 (벤치 < 0.1ms, 무시 가능)
  • 테스트 케이스 증가
